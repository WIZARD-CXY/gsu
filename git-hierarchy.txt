
运行以下的命令，创建了一个空的 Git 仓库之后：

$ mkdir -v test
$ cd test
$ git init

这时，当前目录下生成了一个隐藏目录“.git”，这个目录的内如如下：

branches/    新版本的 Git 不再使用
config       包含了项目特有的配置选项
description  仅供 GitWeb 程序使用
HEAD         指向当前分支
hooks/       包住了客户端或服务端钩子脚本
info/        保存了一份不希望在 .gitignore 文件中管理的忽略模式的全局可执行文件
objects/     存储所有数据内容
refs/        存储指向数据（分支）的提交对象的指针

新版本的 Git 不再使用 branches 目录，description 文件仅供 GitWeb 程序使用。
config 文件包含了项目特有的配置选项，info 目录保存了一份不希望在 .gitignore 文件
中管理的忽略模式的全局可执行文件。hooks 目录包住了客户端或服务端钩子脚本。

如果往仓库中提交了内容，则仓库会发生变化：

$ echo "hello" >> hello
$ git add .
$ git commit -m "1st commit"

这时，当前目录下“.git”目录的内容为：

branches/       新版本的 Git 不再使用
COMMIT_EDITMSG
config          包含了项目特有的配置选项
description     仅供 GitWeb 程序使用
HEAD            指向当前分支
hooks/          包含了客户端或服务端钩子脚本
index           保存了暂存区域信息
info/           保存了一份不希望在.gitignore文件中管理的忽略模式的全局可执行文件
logs/
objects/        存储所有数据内容
refs/           存储指向数据（分支）的提交对象的指针

现在的仓库新增了 2 个文件和 1 个目录：

COMMIT_EDITMSG  最后一次提交的说明信息
index           仓库的索引文件，通常不会出现在纯仓库
logs/           记录对引用的修改

当你的仓库经历了拉取、合并和恢复以后，在 .git 中将会出现以下的文件：

HEAD            指向当前分支
FETCH_HEAD      记录最后一次使用 git fetch 得到的远程仓库的分支
ORIG_HEAD       记录 HEAD 的前一个位置，由更新许多提交对象的命令创建
MERGE_HEAD      记录合并时的提交对象
